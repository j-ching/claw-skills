# 抖音科普文章写作助手 v1.0

## 核心价值

把复杂的技术概念翻译成大白话，让普通用户刷到就能看懂、愿意看完、忍不住分享。

## 与公众号 skill 的关键区别

| | 公众号 | 抖音科普 |
|---|---|---|
| 字数 | 3000-8000 字 | 800-2000 字 |
| 代码 | 可有 1-2 个关键片段 | 完全不出现 |
| 实操步骤 | 有 | 无 |
| 调性 | 专业 + 实用 | 趣味 + 好奇心 |
| 结构 | SCQA 完整模型 | 钩子 → 解释 → 升华 |
| 受众 | 开发者/技术从业者 | 泛科技兴趣人群 |

## 工作流程

```
用户需求 → 选题确定 → 内容规划 → 文章生成 → 配图建议 → 质量检查
```

### 步骤 1：确定题材方向

**方式 A：用户指定主题** — 确认技术领域、目标受众层次（小白/进阶）、内容调性

**方式 B：自动选择热点** — 优先渠道：
- Hacker News 首页、Twitter/X 科技圈热议
- arXiv 高引用论文、GitHub Trending
- 国内：知乎热榜、微博科技热搜

**选题优先级**：
- ⭐⭐⭐⭐⭐ AI 突破/大模型新闻（普通人也能感知的影响）
- ⭐⭐⭐⭐⭐ 反常识科技事实（"你以为...其实..."）
- ⭐⭐⭐⭐ 新技术概念解读（用生活类比解释）
- ⭐⭐⭐ 科技史/人物故事（有戏剧性）
- ⭐⭐ 行业趋势（和日常生活有关联的）

**选题范围**：大语言模型 / AI Agent / 多模态 / 量子计算 / 芯片 / 网络安全 / 航天科技 / 生物技术 / 新能源 / 机器人

详见 [references/topic-selection.md](references/topic-selection.md)。

### 步骤 2：生成文章

采用「钩子 → 解释 → 升华」三段式结构：

**1. 钩子（开头 100-200 字）**
- 用反常识、悬念、或生活场景抓住注意力
- 前 3 秒决定用户是否划走
- 详见 [references/hooks.md](references/hooks.md)

**2. 解释（主体 500-1500 字）**
- 用类比和生活例子解释技术概念
- 每段不超过 3-4 行（适配手机阅读）
- 适当使用 emoji 增强可读性
- 可用流程图/对比表辅助理解
- **绝对不出现代码片段、配置教程、实操步骤**

**3. 升华（结尾 100-300 字）**
- 联系未来趋势或日常生活影响
- 用一个令人回味的观点收尾
- 引导评论互动（提问式结尾）

详见 [references/content-structure.md](references/content-structure.md)。

### 步骤 3：标题优化

抖音标题特点：短、狠、准
- 15-25 字最佳
- 制造好奇心缺口
- 可以用问号、感叹号
- 避免过度标题党（内容要能接住）

详见 [references/hooks.md](references/hooks.md) 中的标题公式。

### 步骤 4：配图建议

抖音图文需要视觉冲击力：
- 建议 3-6 张图
- 比例 9:16（竖屏）或 3:4
- 第一张图必须有视觉钩子（大字标题 / 对比图 / 震惊图）
- 技术概念用信息图/流程图代替文字

有图片生成工具时用工具生成，否则提供配图 Prompt。

### 步骤 5：质量检查

**必须通过**：
- 真实性核查（事实/数据/时间/归属）
- 无代码、无配置、无实操步骤
- 语言通俗，初中学历能看懂
- 段落简短，手机阅读不累
- 有明确的"Wow 时刻"

详见 [references/quality-check.md](references/quality-check.md)。

## 输出规范

- **字数**：800-2000 字，阅读 2-5 分钟
- **段落**：每段 ≤4 行，段间空一行
- **配图**：3-6 张（9:16 竖屏比例）
- **标题**：15-25 字
- **emoji**：适度使用，增强节奏感
- **禁止**：代码片段、配置教程、实操步骤、专业术语堆砌

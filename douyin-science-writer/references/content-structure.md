# 抖音科普文章结构与写作规范

## 核心结构：钩子 → 解释 → 升华

### 一、钩子（开头 100-200 字）

目标：让读者在 3 秒内决定"我要看完"

**技巧 1：反常识开头**
```
你知道吗？现在你手机里的计算能力，比 1969 年送人类登月的 NASA 计算机强了 10 万倍。

但有意思的是，那个「落后」的计算机，完成了人类历史上最伟大的科技壮举。

这是为什么？
```

**技巧 2：悬念开头**
```
有一家科技公司，每年亏 100 亿，但所有人都说它值钱。

它不生产手机，不造汽车，也不做社交软件。

它只做一件事，却可能改变每个人的生活...
```

**技巧 3：生活场景开头**
```
每天早上醒来第一件事是什么？

对很多人来说：拿起手机，刷一下。

但你有没有想过——为什么短视频就是让人停不下来？

不是因为你自制力差。而是背后的算法，比你还了解你的大脑。
```

**技巧 4：提问开头**
```
如果告诉你，有一个东西能在 3 秒钟读完所有人类写过的书，你信吗？

这不是科幻，这是已经发生的事。
```

### 二、解释（主体 500-1500 字）

目标：把一个技术概念讲清楚，让初中学历的人也能听懂

#### 段落写作规范

- 每段 1-4 行，手机阅读不累
- 段与段之间空一行
- 每 3-4 段一个小标题
- 重点句子加粗
- 适度使用 emoji（🔥 💡 ⚡ 🤯 🚀）

#### 类比技巧

**原则**：用读者已有的认知去解释新概念

| 技术概念 | 生活类比 |
|----------|----------|
| 大语言模型 | 超级学霸，读过互联网上几乎所有文字 |
| 神经网络 | 人脑神经元的工作方式 |
| 向量搜索 | 一个知道你想什么的图书管理员 |
| 强化学习 | 训狗：做对了给奖励，做错了不管 |
| 区块链 | 全村人一起记账，谁改不了 |
| 云计算 | 不自己买发电机，而是用电网的电 |
| API 接口 | 餐厅的服务员——你点菜，厨房做菜 |
| 机器学习 | 不是教规则，而是让机器自己找规律 |

#### 对比/表格使用

用对比帮助理解：

```
传统搜索 vs AI 搜索：

传统搜索：你输入关键词 → 它找包含这些词的网页
         像查字典——必须知道 exact 的字

AI 搜索：你说一句话 → 它理解你的意思然后回答
        像问朋友——你说啥它都能接
```

#### 信息图/流程图

用简单的字符流程图：

```
AI 是怎么学会说话的？

海量文字数据 → 训练模型（找规律） → 生成回答 → 人类反馈 → 继续改进
    📚            🧠                💬           👤          🔄
```

### 三、升华（结尾 100-300 字）

目标：让读者看完有"获得感"，想转发或评论

**技巧 1：展望式**
```
也许 5 年后回头看，2025 年就是 AI 的「iPhone 时刻」。

就像 2007 年没人想到手机会改变世界一样，
今天的我们，也正在见证一场更大的变革。

你觉得呢？AI 会让生活更好，还是更焦虑？评论区聊聊 👇
```

**技巧 2：反思式**
```
所以，与其担心 AI 会不会取代我们，
不如想想：有了 AI 的帮助，我们能做到什么以前做不到的事？

技术从来不是答案，用技术的人才是。
```

**技巧 3：互动式**
```
你觉得未来 10 年，AI 会最先改变哪个行业？

A. 教育  B. 医疗  C. 交通  D. 娱乐

评论区告诉我你的答案，顺便说说为什么 🤔
```

## 文章长度指导

- **短文（800-1200 字）**：单一概念解释，1-2 个类比
- **中篇（1200-1800 字）**：概念 + 案例 + 影响分析
- **长文（1800-2000 字）**：多概念串联，有叙事线索

超过 2000 字建议拆成上下篇。

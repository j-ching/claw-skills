### 标题检查清单

- [ ] 字数在 20-26 字之间（最优长度）
- [ ] 包含具体数字（提升可信度）
- [ ] 突出核心价值或收益
- [ ] 制造好奇心或紧迫感
- [ ] 避免标题党（内容要能兑现承诺）
- [ ] 包含目标读者关键词

## 内容结构（SCQA 模型，占流量权重 30%）

```
1. 引言（痛点引入，300 字）
2. 背景知识（500 字）
3. 核心方案（2000 字，分 3-5 个小节）
4. 实操步骤（1500 字，带代码）
5. 性能对比（500 字，数据表格）
6. 常见问题（500 字，FAQ）
7. 总结与延伸（200 字）
8. 结尾引导（行动号召 + 讨论话题）
```

### 结构检查清单

- [ ] 小标题清晰（每 500-800 字一个小标题）
- [ ] 段落简短（每段不超过 5 行）
- [ ] 重点突出（关键句子加粗）
- [ ] 代码块有注释
- [ ] 表格对比清晰
- [ ] 有完整的 FAQ 部分
- [ ] 结尾有行动号召

## 表达优化（占流量权重 10%）

### 1. 类比法（最适合技术科普）

❌ 专业表述："向量数据库通过嵌入模型将文本映射到高维空间"

✅ 类比表述：
```
向量数据库就像一个超级图书管理员：

传统搜索 = 按书名找书（字面匹配）
你说"哈利波特"，它只找书名里有"哈利波特"的书

向量搜索 = 按内容主题找书（语义理解）
你说"魔法学校的故事"，它能找到哈利波特、指环王、纳尼亚传奇
```

### 2. 图解法

用流程图代替文字描述：
```
用户提问 → 意图识别 → 任务分配 → 执行处理 → 结果返回
```

### 3. 案例法

每个技术点都配一个实际案例：
- **电商场景**：如何用 Agent 处理退换货
- **金融场景**：如何用 Agent 做风险评估
- **教育场景**：如何用 Agent 批改作业

### 4. 对话法

用问答形式替代单向讲解：
```
问：CoT 和 ReAct 有什么区别？
答：想象你在解数学题...

问：那什么时候用哪个？
答：看情况...
```

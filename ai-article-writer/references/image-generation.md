# 视觉内容生成规范

## 封面图生成（强制要求）

**⚠️ 每篇文章必须有一张封面图**，尺寸 1536x1024（3:2），蓝紫渐变背景 (#667eea → #764ba2)。

### 生成方式

**如果有图片生成工具**（如 DALL-E、Midjourney API 等）：
1. 文章完成后提取主题、关键词、视觉元素
2. 用下方 Prompt 模板构建提示词
3. 生成后保存到 `images/` 目录
4. 在文章开头引用：`![标题](./images/{文件名}.png)`

**如果没有图片生成工具**：
1. 为每篇文章构建完整的封面 Prompt（见下方模板）
2. 将所有 Prompt 保存到 `cover-images.md` 文件
3. 提示用户用外部工具（Midjourney / DALL-E / Canva）生成

### Prompt 构建模板
```
技术博客封面图，{文章主题}，
包含{核心技术元素1}、{核心技术元素2}、{核心技术元素3}，
现代简约科技风格，蓝紫色渐变背景 (#667eea → #764ba2)，
专业感强，适合 AI/技术开发者阅读，
高质量，详细，4k
```

### 注意事项
- 图片内容必须与文章主题强相关
- 避免生成具体人物肖像（用抽象图标/图形代替）
- 配色统一使用蓝紫色系
- 每篇文章的封面 Prompt 需独特化，不能雷同

## 头图规范（900x383px, 2.35:1）

**风格**：专业、简洁、科技感

**内容**：架构图、流程图、对比图

**配色**：品牌色统一（蓝紫色系 #667eea → #764ba2）

**格式**：PNG（透明背景或渐变）

### Prompt 模板

```
技术博客头图，{主题} 示意图，
包含{元素1}、{元素2}、{元素3}，
现代简约风格，蓝紫色渐变背景，
专业感强，适合开发者阅读，
分辨率 900x383 像素
```

## 文中配图（600x400px, 3:2）

### 类型

- **架构图**：系统组件关系
- **流程图**：执行步骤顺序
- **对比图**：优化前后差异
- **数据图**：性能测试结果

### 风格

- 简洁线条
- 清晰标注
- 统一配色
- 适当留白

## 代码截图规范

### 工具推荐

- **Carbon** (carbon.now.sh)：最美代码截图工具
- **Ray.so**：Vercel 出品，简约风格
- **CodeSnap**：VS Code 插件，快速出图

### 样式要求

- 字号：14-16px（手机清晰可见）
- 主题：One Dark / GitHub Light
- 行号：显示（方便定位）
- 圆角：8-12px
- 阴影：轻微投影

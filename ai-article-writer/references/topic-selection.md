# 公众号用户画像与选题策略

## 公众号平台特性

**核心优势**：
- 私域流量：粉丝订阅制，内容直达用户
- 社交推荐：朋友圈 + 看一看，裂变传播
- 长尾效应：优质内容持续带来长尾流量
- 变现路径：付费阅读 + 广告 + 私域转化

**算法机制**：
```
流量来源 = 订阅推送(40%) + 朋友圈(30%) + 看一看(20%) + 其他(10%)

推荐权重:
- 打开率：>15% 进入流量池
- 读完率：>70% 触发看一看推荐
- 分享率：>5% 触发朋友圈扩散
- 互动率：>3% 提升账号权重
```

**流量层级**：
```
第一级：订阅用户（基础流量）
 ↓ 打开率>15%
第二级：朋友圈（2-5 倍放大）
 ↓ 分享率>5%
第三级：看一看（5-10 倍放大）
 ↓ 读完率>70%
第四级：搜一搜（长尾流量）
```

## 用户深度画像

**年龄分布**：
- 25-35 岁：45%（职场上升期）
- 35-45 岁：35%（管理者/决策者）
- 18-25 岁：15%（学生/职场新人）
- 45 岁+：5%（资深专家）

**职业分布**：
- 互联网/科技：35%
- 金融/咨询：20%
- 教育/培训：15%
- 传统行业数字化：20%
- 其他：10%

**用户特征**：
✓ 有明确学习目的和成长需求
✓ 愿意为高质量内容付费
✓ 重视专业性和可信度
✓ 决策理性，但容易被数据说服
✓ 时间碎片化，偏好高效学习
✓ 有较强的社交分享意愿（建立专业形象）

**核心痛点**：
✗ 信息过载，需要筛选和总结
✗ 技术更新快，担心落后
✗ 缺乏系统性学习路径
✗ 实战经验不足，需要案例指导
✗ 职业发展焦虑，需要技能提升

**内容偏好**：
★ 系统性教程（从入门到精通）
★ 行业洞察和趋势分析
★ 实战案例和最佳实践
★ 工具和方法论
★ 大厂经验和技术分享
★ 性能对比和数据分析

## 阅读量关键指标

**核心公式**：`阅读量 = 推送人数 × 打开率 + 朋友圈点击 + 看一看推荐`

| 指标 | 及格 | 良好 | 优秀 | 爆款 |
|------|------|------|------|------|
| 打开率 | >5% | >10% | >15% | >20% |
| 读完率 | >30% | >50% | >70% | >85% |
| 分享率 | >1% | >3% | >5% | >10% |
| 互动率 | >0.5% | >1% | >2% | >5% |

## 选题策略（基于真实数据）

### 高阅读量内容类型（优先选择）

**✅ AI 框架对比类**（阅读占比 12.05%，排名第 1）
- 示例："Agent 开发框架对比：LangChain、AutoGen、Agno 哪个更适合你？"
- 写法：多框架横向对比 + 性能数据 + 选型建议

**✅ 新框架基础功能介绍**（阅读占比 11.79%，排名第 2）
- 示例："字节开源的 AI Agent 框架，上线 30 天狂揽 5.9 万 Star！"
- 写法：项目亮点 + 快速上手 + 核心特性概览

**✅ 快速上手/入门教程**（阅读占比 8.97%，排名第 3）
- 示例："10 分钟搭建 OpenClaw 开发环境"
- 写法：步骤清晰 + 时间短承诺 + 预期结果明确

**✅ Multi-Agent/系统架构概览**（阅读占比 6.67%，排名第 4）
- 示例："Multi-Agent 系统：让 AI 团队协作性能提升 300%"
- 写法：架构图解 + 场景举例 + 效果对比

### 低阅读量内容类型（减少或简化）

**❌ 配置类详解** — 简化为"3 步快速配置"，重点讲效果而非配置细节

**❌ 代码实现细节** — 用流程图/架构图代替代码，只展示核心逻辑

**❌ 深度技术原理** — 用类比和场景故事引入，降低理解门槛

### 内容深浅度平衡原则

**黄金比例**：70% 浅显易懂 + 30% 技术深度

| 文章部分 | 占比 | 深浅度 |
|---------|------|--------|
| 开篇 | 20% | 浅显：痛点场景 + 收益承诺 |
| 核心概念 | 30% | 浅显：类比解释 + 图解说明 |
| 实战演示 | 30% | 中等：关键步骤 + 效果展示 |
| 进阶技巧 | 20% | 深度：深度优化 + 最佳实践 |

### 选题优先级矩阵

| | 低门槛 | 高门槛 |
|--|--------|--------|
| 高吸引力 | **优先写**（框架对比、新框架介绍、快速上手） | 简化后写（用类比降低门槛） |
| 低吸引力 | 偶尔写（保持内容多样性） | **避免写**（或拆分成系列浅显文章） |

## 确定题材的提问模板

当用户指定主题时，询问以下问题：

1. **技术领域**：LLM 应用开发 / Agent 构建 / RAG 系统 / 多模态 AI / 其他
2. **目标读者**：开发者（技术深度）/ 产品经理（应用场景）/ 企业决策者（商业价值）/ 学生/初学者（入门学习）
3. **内容调性**：专业严谨 / 轻松实用 / 数据驱动 / 故事化
4. **实操要求**：每篇都要可运行代码 / 部分需要 / 不需要

# 公众号用户画像与选题策略

## 公众号平台特性

**核心优势**：
- 私域流量：粉丝订阅制，内容直达用户
- 社交推荐：朋友圈 + 看一看，裂变传播
- 长尾效应：优质内容持续带来长尾流量
- 变现路径：付费阅读 + 广告 + 私域转化

**算法机制**：
```
流量来源 = 订阅推送(40%) + 朋友圈(30%) + 看一看(20%) + 其他(10%)

推荐权重:
- 打开率：>15% 进入流量池
- 读完率：>70% 触发看一看推荐
- 分享率：>5% 触发朋友圈扩散
- 互动率：>3% 提升账号权重
```

**流量层级**：
```
第一级：订阅用户（基础流量）
 ↓ 打开率>15%
第二级：朋友圈（2-5 倍放大）
 ↓ 分享率>5%
第三级：看一看（5-10 倍放大）
 ↓ 读完率>70%
第四级：搜一搜（长尾流量）
```

## 用户深度画像

**年龄分布**：
- 25-35 岁：45%（职场上升期）
- 35-45 岁：35%（管理者/决策者）
- 18-25 岁：15%（学生/职场新人）
- 45 岁+：5%（资深专家）

**职业分布**：
- 互联网/科技：35%
- 金融/咨询：20%
- 教育/培训：15%
- 传统行业数字化：20%
- 其他：10%

**用户特征**：
✓ 有明确学习目的和成长需求
✓ 愿意为高质量内容付费
✓ 重视专业性和可信度
✓ 决策理性，但容易被数据说服
✓ 时间碎片化，偏好高效学习
✓ 有较强的社交分享意愿（建立专业形象）

**核心痛点**：
✗ 信息过载，需要筛选和总结
✗ 技术更新快，担心落后
✗ 缺乏系统性学习路径
✗ 实战经验不足，需要案例指导
✗ 职业发展焦虑，需要技能提升

**内容偏好**：
★ 系统性教程（从入门到精通）
★ 行业洞察和趋势分析
★ 实战案例和最佳实践
★ 工具和方法论
★ 大厂经验和技术分享
★ 性能对比和数据分析

## 阅读量关键指标

**核心公式**：`阅读量 = 推送人数 × 打开率 + 朋友圈点击 + 看一看推荐`

| 指标 | 及格 | 良好 | 优秀 | 爆款 |
|------|------|------|------|------|
| 打开率 | >5% | >10% | >15% | >20% |
| 读完率 | >30% | >50% | >70% | >85% |
| 分享率 | >1% | >3% | >5% | >10% |
| 互动率 | >0.5% | >1% | >2% | >5% |

## 选题策略（基于互联网热度）

### 热点来源渠道（优先级排序）

**海外渠道（优先）**：
1. **Hacker News** — 技术社区热度，首页/高评论帖子
2. **arXiv** — 最新 AI/ML 论文，按引用和讨论热度
3. **GitHub Trending** — 开源项目 Stars 增长速度
4. **Product Hunt** — 新产品发布，社区投票
5. **Twitter/X** — 技术讨论、官宣、热议话题

**国内渠道（补充）**：
6. **知乎** — 技术话题讨论热度
7. **微信公众号** — 行业热点文章
8. **掘金/V2EX** — 开发者讨论

### 选题优先级（按热度排序）

| 优先级 | 内容类型 | 热度来源 | 示例 |
|--------|---------|---------|------|
| ⭐⭐⭐⭐⭐ | 热门事件/重磅发布 | Hacker News 首页、Twitter 热议 | OpenAI 发布 GPT-5、Claude 3.5 上线 |
| ⭐⭐⭐⭐⭐ | 框架对比 | GitHub Stars、社区讨论 | LangChain vs AutoGen vs Agno 哪个更适合你？ |
| ⭐⭐⭐⭐ | 新框架/工具介绍 | GitHub Trending、Product Hunt | 字节开源 Eino，30 天 5.9 万 Star |
| ⭐⭐⭐⭐ | 快速上手教程 | 搜索需求、社区问答 | 10 分钟搭建 X 开发环境 |
| ⭐⭐⭐ | 架构概览 | 技术博客引用 | Multi-Agent 系统设计 |
| ⭐⭐ | 原理解析 | arXiv 论文引用 | Transformer 注意力机制详解 |
| ⭐ | 配置细节 | 文档搜索 | Nginx 配置详解 |

### 选题范围扩展

**AI/大模型核心领域**：
- 大语言模型（LLM）：GPT、Claude、Gemini、开源模型等
- AI Agent：框架、架构、工具链、评测
- RAG 系统：检索增强生成、向量数据库、知识图谱
- 多模态 AI：视觉、语音、视频生成
- 强化学习（Agentic RL）：RLHF、DPO、GRPO、RL 与 LLM 结合
- 深度学习：Transformer、CNN、RNN、注意力机制、模型架构
- 模型训练与微调：LoRA、QLoRA、PEFT、全参微调、分布式训练
- AI 评测与排行榜：基准测试、Agent Evals、能力评估

**周边工具与框架**：
- AI 开发框架：LangChain、AutoGen、CrewAI、Semantic Kernel
- 模型训练框架：PyTorch、TensorFlow、JAX、Megatron-LM
- 分布式训练：Ray、DeepSpeed、FSDP、Tensor Parallel
- 模型部署：vLLM、TGI、Triton、ONNX
- 向量数据库：Pinecone、Weaviate、Milvus、FAISS
- MLOps 工具：MLflow、Weights & Biases、Kubeflow

### 热度判断依据

- **Hacker News**：排名、评论数、点赞数
- **GitHub**：Stars 增长速度（日增/周增）、Fork 数
- **Twitter/X**：转发、点赞、评论、话题热度
- **知乎/掘金**：浏览量、赞同数、评论数
- **arXiv**：引用数、社交媒体讨论热度

### 低优先级内容（减少或简化）

**❌ 配置类详解** — 简化为"3 步快速配置"，重点讲效果而非配置细节

**❌ 代码实现细节** — 用流程图/架构图代替代码，只展示核心逻辑

**❌ 深度技术原理** — 用类比和场景故事引入，降低理解门槛

### 内容深浅度平衡原则

**黄金比例**：70% 浅显易懂 + 30% 技术深度

| 文章部分 | 占比 | 深浅度 |
|---------|------|--------|
| 开篇 | 20% | 浅显：痛点场景 + 收益承诺 |
| 核心概念 | 30% | 浅显：类比解释 + 图解说明 |
| 实战演示 | 30% | 中等：关键步骤 + 效果展示 |
| 进阶技巧 | 20% | 深度：深度优化 + 最佳实践 |

## 确定题材的提问模板

当用户指定主题时，询问以下问题：

1. **技术领域**：
   - 大语言模型 / AI Agent / RAG 系统 / 多模态 AI
   - Agentic RL / 强化学习 / 模型训练与微调
   - 深度学习基础 / 模型架构 / 算法原理
   - 周边框架：PyTorch、TensorFlow、Ray、DeepSpeed、vLLM 等
   - AI 评测 / 排行榜 / 基准测试
2. **目标读者**：开发者（技术深度）/ 产品经理（应用场景）/ 企业决策者（商业价值）/ 学生/初学者（入门学习）
3. **内容调性**：专业严谨 / 轻松实用 / 数据驱动 / 故事化
4. **实操要求**：每篇都要可运行代码 / 部分需要 / 不需要

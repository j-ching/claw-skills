## 核心能力一览

### 🔥 选题：不是猜，是算

```
高阅读量内容类型（真实数据排名）：

1️⃣ AI 框架对比类    阅读占比 12.05%  ← 优先写
2️⃣ 新框架基础介绍    阅读占比 11.79%  ← 优先写
3️⃣ 快速上手教程      阅读占比  8.97%  ← 优先写
4️⃣ Multi-Agent 架构  阅读占比  6.67%  ← 优先写
5️⃣ 配置类详解        排名靠后          ← 简化
6️⃣ 代码实现细节      排名靠后          ← 简化
```

自动从 GitHub Trending、Hacker News、arXiv 等渠道抓取热点，结合真实数据排名，告诉你**这周该写什么**。

### 💥 标题：不是想，是选

7 套经过验证的标题公式，每个场景对应最优解：

```
公式1: 痛点场景 + 解决方案 + 数据支撑
  → 《客服响应慢被投诉？用这个技术让回复速度提升 10 倍！》

公式4: 对比反差 + 具体数字 + 利益诱惑
  → 《同样是用 AI，为什么别人效率是你的 5 倍？秘密在这里》

公式6: 失败教训 + 避坑指南 + 情感共鸣
  → 《花了 10 万买来的教训：这 5 个 AI 坑千万别踩》
```

### 🧠 开篇：黄金 3 秒法则

前 3 段决定读者是否关掉文章。5 种模板，对应 5 种场景：

- **痛点场景化** — 最适合技术教程
- **故事引入** — 最适合案例分享
- **数据冲击** — 最适合性能优化
- **反常识观点** — 最适合趋势分析
- **身份认同** — 最适合经验总结

### 📐 SCQA 内容结构

每篇文章自动套用麦肯锡级逻辑框架：

```
Situation（情境）→ Complication（冲突）→ Question（问题）→ Answer（答案）

1. 引言（300 字）    痛点引入，抓住注意力
2. 背景知识（500 字） 核心概念解释
3. 核心方案（2000 字）3-5 个小节，每节都有小标题
4. 实操步骤（1500 字）带代码，可运行
5. 性能对比（500 字） 数据表格，一目了然
6. 常见问题（500 字） FAQ，解决疑惑
7. 总结延伸（200 字） 回顾核心要点
8. 结尾引导          行动号召 + 讨论话题
```

### 🛡️ 五层 Fake News 检测

技术文章最怕什么？瞎编数据。

这个 Skill 内置了五层检测机制：

```
❌ 完全捏造   → 必须删除
❌ 张冠李戴   → 必须修正
❌ 时间错乱   → 必须修正
❌ 数据造假   → 必须删除
❌ 断章取义   → 必须修正

✅ 互联网有迹可循    → 保留
✅ 无法证实但合理    → 保留
❌ 凭空捏造无中生有   → 删除
```

### 🎨 AI 封面图生成

每篇文章自动构建封面 Prompt，统一蓝紫渐变配色 (#667eea → #764ba2)，保持专业科技感。
